# Distilled — Quality Classifier

A small local model that scores how useful a short text snippet would be for teaching a language model to write in a user's voice. Labeled by Opus 4.7, distilled into a scikit-learn TF-IDF + LogisticRegression `Pipeline` shipped as `quality-classifier.pkl`.

See `.claude/skills/distillation-pipeline/` for the operational recipe. This README is the contract.

## Input / output

Input: UTF-8 text, ≤ 1,000 characters.

Output:

```json
{"score": 0.73, "reason": "voice-bearing but short; single complete thought"}
```

- `score ∈ [0, 1]` — higher means better training signal.
- `reason` — short free-text string. Informational; not used by the filter.

## Thresholds (used by `KilnCore`)

- `score ≥ 0.70` → keep for SFT.
- `0.40 ≤ score < 0.70` → keep only as a `chosen` example in DPO.
- `score < 0.40` → discard.

Thresholds are calibrated at distillation time, not tuned at runtime.

## Training data origin

- Source: a 10,000-row mixed corpus sampled from user-provided writing (PII-scrubbed), public-domain high-quality writing, and synthetic low-quality scrape artifacts.
- Labels: Opus 4.7 scores + reasons, produced by `scripts/opus-distill/run.py --component quality-classifier`.
- Split: 80/10/10 stratified by source bucket.

## Artifacts

- `packages/kiln_trainer/artifacts/quality-classifier.pkl` — sklearn `Pipeline` (TF-IDF + LogisticRegression), 567 KB. The runtime ships this directly; the SPEC §7.2 phrasing of `model.mlmodel` predates M9.C — there is no CoreML conversion in the path today.
- `manifest.json` — version, git SHA, Opus teacher id, training counts, metrics, ship-bar status. Generated by `scripts/build_distilled_manifests.py`.
- `raw_labels.jsonl` — gitignored; produced by the corpus-builder Managed Agent.

## Eval (ship bar and current)

| Metric | Bar | Current |
|---|---|---|
| Test accuracy | ≥ 0.80 | **0.99** |
| Train accuracy | informational | 0.9975 |
| Disagreement w/ Opus (margin) | informational | 0.01 |
| Label count | informational | 1500 (1200 train / 300 test) |

F1/precision/recall aren't computed at training time — the binary
≥ 0.5 framing has near-balanced classes in this corpus and test
accuracy is the headline metric. The auditor's recommended ship-bar
re-framing (test accuracy ≥ 0.80) is what `manifest.json` records.

Below bar → do not ship; re-distill or reshape labels.

## Loading in Swift

The KilnCore consumer is `SubprocessDistilledClassifierRunner`, which
spawns the Python sidecar's `classify --mode quality --artifact
<path>.pkl` rather than loading the artifact in-process. Swift never
unpickles directly.

## Versioning

Pinned via `manifest.json`'s `version` field. The pickle's payload
also carries a `version` key; the sidecar refuses to load a version
mismatch — fail loud.
